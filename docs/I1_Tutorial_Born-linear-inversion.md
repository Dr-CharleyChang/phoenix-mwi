# I1 Tutorial: Born linear inversion — the first χ-map

> **How to use this tutorial:** read it once for the whole arc, then implement it yourself against the Phase-0 spine, returning here when stuck. It gives the physics, the discrete linear model, the algorithm, a function/class checklist, and self-tests — **not** a full solution. When the I1 tests go green you get the project's first *image*: a recovered contrast map χ̂ of an unknown object.

> **Where I1 sits.** F1/F2 solved the *forward* problem (known object → fields). I1 turns around: *measured fields → unknown object*. We do the **linear** version first (Born). **DBIM (I2) is just I1 run in an outer loop**, re-linearizing around the latest estimate — so everything here is the engine DBIM will drive. Build the engine before the loop.

---

## 0. Goal and acceptance criteria

**Goal:** from multi-view scattered-field measurements of a *weak* dielectric scatterer, reconstruct its contrast map χ̂ by solving a single **regularized linear least-squares** problem — using the matrix-free Born operator `A_op` and its adjoint `AH_op` (already in `operators.py`) plus a Krylov least-squares solver (LSQR/LSMR).

**Acceptance criteria (I1 passes when all hold):**
1. **Adjoint test:** ⟨A χ, u⟩ = ⟨χ, Aᴴ u⟩ to ~1e-10 on random vectors (without this, LSQR/CGLS cannot converge — it is the single most important unit test).
2. **Inverse crime:** when the data are generated by the Born operator itself (`d = A χ_true`), the inversion recovers χ_true to <1e-3 (pure linear-algebra check).
3. **Physical recovery:** with data from the *full* forward solver (`MoM2D`) on a weak scatterer (ε_r ≈ 1.1), χ̂ is localized at the true support, real part the right sign, relative error below a set threshold (Born has model error — the threshold is loose).
4. A **χ-map figure**: true vs reconstructed (real part), side by side.

---

## 1. From forward to inverse — the picture

The forward problem is a function `F`: object χ ↦ data d (fields at receivers). Inversion asks for `F⁻¹`: data d ↦ object χ. Two hard facts make this not a simple "divide":

- **It's ill-posed.** Many different χ produce nearly the same data (the map loses information — high-spatial-frequency detail radiates weakly). So we cannot just solve; we must *regularize* (add a preference for "reasonable" solutions).
- **It's nonlinear in general.** Recall the Lippmann–Schwinger equation (F1 §2.4): the field inside the object depends on χ *and* on the field, which depends on χ. χ and the field multiply. **The Born approximation cuts this knot by linearizing.**

---

## 2. The Born approximation — why it becomes linear

The exact scattered field at receiver `r_r` for incidence `i` is

$$E^{\text{sc}}_i(\mathbf r_r)=k_b^2\!\int_S G(\mathbf r_r,\mathbf r')\,\chi(\mathbf r')\,E_i(\mathbf r')\,dS',$$

where $E_i$ is the **total** field inside the object (unknown, χ-dependent). For a **weak** scatterer the object barely perturbs the field, so $E_i \approx E^{\text{inc}}_i$ (the known incident field). Substitute:

$$\boxed{\,E^{\text{sc}}_i(\mathbf r_r)\approx k_b^2\!\int_S G(\mathbf r_r,\mathbf r')\,\chi(\mathbf r')\,E^{\text{inc}}_i(\mathbf r')\,dS'\,}$$

Now the only unknown is χ, and it appears **linearly**. That is the entire trick: replace the unknown interior field by the known incident field. Valid when |χ| and the object are small (the regime where F1's "weak scatterer" tests live).

---

## 3. The discrete linear model `d = A χ`

Discretize the integral on the same grid as the forward solver (cells at `centers`, area `dS`). For one incidence `i`, the data at the `M` receivers is

$$d^{(i)}_m = k_b^2\,dS\sum_{n} G(\mathbf r_m,\mathbf r_n)\,E^{\text{inc}}_i(\mathbf r_n)\,\chi_n = \big(A^{(i)}\chi\big)_m,\qquad A^{(i)}_{mn}=k_b^2\,dS\,G_{\text{tr}}[m,n]\,E^{\text{inc}}_i[n].$$

- `G_tr` is the **receiver×cell** Green matrix, `G_tr[m,n] = (1/4j) H₀⁽²⁾(k_b |r_m − r_n|)` (receivers are outside the object, so no self-cell singularity — reuse `green_2d`).
- This is **exactly** the operator already in `operators.py`: `A_op(χ, E_inc_i, G_tr, k_b, dS) = k_b² · dS · G_tr @ (E_inc_i * χ)`. Read it: weight χ by the incident field per cell (`E_inc_i * χ`), then propagate to the receivers (`G_tr @ …`).

**One incidence is not enough.** `M` receivers give `M` numbers; the image has `N ≫ M` unknowns → hopelessly underdetermined, and one "view" sees the object from one angle only. So we use **multiple incidences** (plane waves from `N_v` directions). Stack them:

$$\mathbf A=\begin{bmatrix}A^{(1)}\\\vdots\\A^{(N_v)}\end{bmatrix},\quad \mathbf d=\begin{bmatrix}d^{(1)}\\\vdots\\d^{(N_v)}\end{bmatrix},\qquad \mathbf A\,\chi=\mathbf d.$$

`A` is `(N_v·M) × N`. We never build it densely — `A_op`/`AH_op` give matrix-free matvecs.

---

## 4. Ill-posedness and regularization (Tikhonov)

`A χ = d` is ill-posed (and noisy in practice), so solving it exactly amplifies noise into garbage. Instead solve the **regularized least-squares** problem

$$\min_{\chi}\ \lVert \mathbf A\chi-\mathbf d\rVert_2^2+\mu\,\lVert\chi\rVert_2^2,$$

the **Tikhonov** form. The `μ‖χ‖²` term penalizes wild solutions; `μ` trades data-fit against smoothness/energy. The normal equations are `(AᴴA + μI)χ = Aᴴd`, but we **do not** form `AᴴA`; we hand `A_op`/`AH_op` to a Krylov least-squares solver:
- `scipy.sparse.linalg.lsmr` (or `lsqr`) takes a `LinearOperator` with `matvec`=A and `rmatvec`=Aᴴ, plus `damp=√μ` for Tikhonov. (CGLS is the same idea; LSMR is its robust cousin.)

> Choosing μ is its own craft (L-curve, discrepancy principle). For I1, a fixed small μ is fine; expose it as a knob.

---

## 5. The adjoint, and why the adjoint test is sacred（From Here）

LSQR/LSMR/CGLS rely on `rmatvec` being the **exact mathematical adjoint** Aᴴ of `matvec` A, i.e. ⟨Aχ, u⟩ = ⟨χ, Aᴴu⟩ for all χ, u (complex inner product). The adjoint of the per-view operator is

$$\big(A^{(i)}\big)^{H}u=k_b^2\,dS\,\overline{E^{\text{inc}}_i}\odot\big(G_{\text{tr}}^{H}u\big),$$

which is exactly `AH_op(u, E_inc_i, G_tr, k_b, dS)` — back-propagate the receiver residual `u` (`G_trᴴ @ u`), then de-weight by the conjugate incident field. **Physically `AH_op` is back-projection**: smear the residual from the receivers back onto the grid. `AH_op(d)` alone is already a crude image (this is the link to qualitative imaging).

If the adjoint is even slightly wrong, the solver silently fails to converge. So the **adjoint test is the first test you make pass**, before any reconstruction.

For the stacked multiview operator: `A` concatenates per-view outputs; `Aᴴ` splits `u` into per-view blocks, applies each `AH_op`, and **sums** over views.

---

## 6. Generating the synthetic data (honestly)

To test inversion you need data. Two sources, used for two different tests:
- **Inverse-crime data** (criterion 2): `d = A χ_true` using the *Born operator itself*. Inverting this isolates the linear algebra (no model error) → must recover χ_true almost exactly. A pure unit test.
- **Physical data** (criterion 3): run the *full* forward solver `MoM2D` (F1/F2) for each incidence on a weak scatterer, sample the scattered field at the receivers. Now the data contain the real Born *model error* (because the true interior field ≠ incident field), so recovery is approximate — the realistic case.

Geometry: `N_v` plane-wave incidence angles around the object, `M` receivers on a ring (reuse the F1 ring). Single frequency for I1.

## 7. Function / class checklist (what to implement)

Put these in `mwisim/inverse/` (the package stub already exists). Implement behind the Phase-0 `Inverter` interface.

1. **`green_matrix(rx, centers, k_b)` → `G_tr` `(M, N)`** — receiver×cell Green matrix via `green_2d` (broadcast distances; rx outside ⇒ no singularity). *Reuses F1.*
2. **`plane_wave_incidences(centers, k_b, angles)` → `E_inc` `(N_v, N)`** — incident field on the grid for each direction: $E^{\text{inc}}_i(\mathbf r)=e^{-jk_b(\hat{\mathbf k}_i\cdot\mathbf r)}$.
3. **`BornOperator(centers, rx, E_inc_set, k_b, dS)`** — holds geometry; provides:
- `matvec(chi)` → stacked `d` over views (loop `A_op` per view, concatenate).
- `rmatvec(u)` → `chi` (split `u` per view, `AH_op` each, **sum**).
- `as_linear_operator()` → `scipy.sparse.linalg.LinearOperator(shape=(N_v*M, N), matvec, rmatvec, dtype=complex)`.
4. **`BornInverter(Inverter)`** (register `@register("inverter","born")`):
- `__init__(self, mu=1e-2, iter_lim=200)` — knobs in the constructor (PROJECT_PLAN §11.3).
- `reconstruct(self, data, forward=None, x0=None, **kw)` → `(chi, info)`: build `BornOperator` from `data`’s geometry, call `lsmr(op, d, damp=sqrt(mu), maxiter=iter_lim)`, return χ̂ and an info dict (iters, residual, μ). *(`forward` is unused here — it's the hook DBIM/I2 will use to re-solve the field each outer iteration.)*
5. **(helper) `make_born_problem(...)`** — assemble the synthetic `data` dict: `centers, dS, k_b, rx, E_inc_set, d, chi_true`. This is the proto **data schema** (PROJECT_PLAN §11.1); keep the keys clean — Phase 1 formalizes it.

> Reuse, don't reinvent: `A_op`/`AH_op` (operators.py), `green_2d` (green.py), `incident_plane_wave` pattern (mom.py), `MoM2D` (forward/mom2d.py) for physical data, `rel_l2_error` (metrics.py).

## 8. Self-tests (I1.1 – I1.5) — your progress bar

Write these in `tests/test_i1.py`; make them green one at a time.

- **I1.1 — `green_matrix` sanity:** shape `(M, N)`; equal receiver–cell distances give equal entries (depends only on distance).
- **I1.2 — adjoint test (the gate):** random complex `χ` `(N,)` and `u` `(N_v·M,)`: `abs(⟨Aχ,u⟩ − ⟨χ,Aᴴu⟩) / abs(⟨Aχ,u⟩) < 1e-10`. *Make this pass first.*
- **I1.3 — inverse crime:** `d = op.matvec(χ_true)`; reconstruct; `rel_l2_error(χ̂, χ_true) < 1e-3`.
- **I1.4 — physical recovery:** data from `MoM2D` on a weak cylinder (ε_r = 1.1); reconstruct; assert χ̂ peak is at the object center (localization) and `rel_l2_error(χ̂, χ_true)` below a loose threshold (e.g. < 0.6 — Born model error is real; tune once you see it).
- **I1.5 — registry:** `build("inverter","born", mu=1e-2)` returns a working `BornInverter`.

## 10. Pitfalls (most will bite at least once)

- **Wrong adjoint.** If I1.2 fails, nothing else can pass. Check the conjugate in `AH_op` (`conj(E_inc)`) and the Hermitian transpose `G_tr.conj().T`.
- **Forgetting to sum over views in `rmatvec`.** Each view contributes; Aᴴ is a sum, not a concatenation.
- **No regularization (μ = 0).** The solve will chase noise/model-error and look like static. Start with a visible μ and reduce.
- **Born outside its regime.** If you test on a *strong* scatterer, recovery is poor — that's physics, not a bug. Born needs weak contrast; strong contrast is what DBIM (I2) fixes.
- **Real vs complex.** χ is generally complex (lossy media); keep everything complex, and plot the real part for the figure.
- **Grid mismatch.** The inversion grid (where χ lives) and the `E_inc`/`G_tr` cell grid must be the same `centers`; reuse one `Phantom.grid()`.

---

## 11. What I1 unlocks

Once Born inverts, **DBIM (I2)** is a short step: wrap I1 in an outer loop that, each iteration, (a) solves the *full* forward field for the current χ̂ with `MoM2D` (no Born approximation), (b) re-linearizes (updates `E_inc → E_total`, rebuilds the operator), and (c) takes a regularized Born step on the residual. Same `A_op`/`AH_op`, same LSQR — now handling strong contrast. The `forward` argument of `reconstruct` is the hook for exactly that.
